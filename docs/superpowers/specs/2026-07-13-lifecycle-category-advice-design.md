# 完整交易复盘分类建议设计

## 目标

分类优化建议只从完整交易复盘生成。建议必须有足够样本、证据明确、数量少且可以直接执行。

## 聚合规则

- 每个策略来源和币种分类至少累计 8 笔完整交易。
- 同一问题至少出现 3 次，且占该分类有效样本的 30%。
- 排除后续数据不足和结论不明确的交易，不用它们推动策略调整。
- 每个分类只保留证据最强的一项问题，全局最多显示 5 项。
- 相同策略来源、相同问题和相同动作的分类合并为一项，并列出受影响分类与代表币种。

## 问题识别

- 开仓确认不足：亏损、开仓后最大有利空间小、最大不利回撤明显，且平仓后没有证明属于平早。
- 平仓过早：平仓后仍出现超过分类阈值的原方向空间。
- 有效退出、合理盈利、数据不足和混合结论不生成优化建议。

## 输出

每项只展示优先级、受影响分类、有效样本、问题次数与占比、合计盈亏、平均最大有利/不利空间、代表币种、简短结论和一条可执行建议。

## 验收

- 小样本和低占比问题不展示。
- 每类最多一条，全局最多五条。
- Alpha 开仓确认不足输出已确认的同步放量和结构确认建议。
- API 与前端均使用新字段，历史完整交易复盘保持不变。
